## 자바 컴파일 과정

자바컴파일러가 자바 소스코드 파일을 자바 바이트코드로 변환하고 JVM의 클래스 로더로 전달된다.
이후 JVM에서 코드를 실행한다.
바이트 코드 변환을 통해 CPU와 OS로 부터 독립적이다.
## 자바 관련 용어 

JRE = Java Runtime Environment
JDK = Java Development Kit
JRE는 자바 언어나 가상머신등의 도구는 포함하지 않는다. 런타임 내 환경만 포함

JDK는 자바런타임에 자바언어와 도구까지 총칭한다.
## Keywords

자바 Keyword들은 변수명으로 사용할 수 없다.

## 자료형

기본타입 , 레퍼런스 타입 으로 나누어진다.
기본타입 = int , float , boolean ...

레페런스 타입 = 클래스, 인터페이스, 배열

## 변수명(식별자) 관례

클래스명 -> 각 단어 첫글자 대문자
변수명, 메소드명 -> 소문자로 시작 2번째 단어의 첫글자 대문자
상수 -> 모든 글자 대문자

## 원시 데이터 타입

자바의 8가지 기본 타입으로 크기가 정해져 있음.

## WraperClass

자바의 기본형을 정의해놓은 클래스로 ArrayList등을 구현할 때 Class를 템플릿으로 사용하기 때문에 필요하다.
