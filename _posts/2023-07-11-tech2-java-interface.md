자바 인터페이스와 추상클래스

## Java 에서 인터페이스의 존재 이유에 대해


자바 인터페이스는 공통된 속성들을 묶어서 하나의 설계도와 같은 역할을 한다. 일반적인 CS에서 말하는 인터페이스와는 다른 개념이다. C++ , Java에 존재하는 추상클래스와 굉장히 비슷한 역할을 하는데 이미 추상클래스가 존재하는데 왜 자바 인터페이스를 또 만들어야 했던 것일까?

## 추상클래스와 인터페이스의 차이

기본적으로 자바에서 추상클래스는 다중 상속이 금지되어 있다. 다중상속이 주는 강점이 분명히 있음에도 불구하고 다중상속을 금지하는 이유는 <다이아몬드 문제> 때문이다.(다중 상속시에 같은 이름의 메소드가 있을 경우 어떤 메소드인지 구분이 안되는 문제) 이는 추상클래스의 경우 100%추상메소드가 아닌 구현부를 포함한 메소드가 가능하기 때문이다.

반면 인터페이스는 100% 추상메소드로만 구성이 가능하기 때문에 이런문제가 발생하지 않는다. 즉 다중상속이 가능해진다. (다중상속을 받고 같은 이름의 메소드가 있다고 하더라도 구현부가 비어있기 때문에 다이아몬드 문제가 발생되지 않는다)

## 실제 활용

서로 관련도가 높을 경우 추상클래스 , 관련도가 높지 않은 상태에서 연관관계를 맺을 경우 인터페이스로 사용한다.
