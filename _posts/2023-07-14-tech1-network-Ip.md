## IPv4

IPv4 는 3자리 숫자가 4마디로 표기되는 방식이다. 각 마디는 옥텟(octet)이라고 부른다.

첫 번째 옥탯의 범위에 따라 A,B,C,D클래스 등으로 나뉜다. A(0~127) B(128~191) C(192~223) D(224~239) A,B,C가 주로 사용되며 순차적으로 대규모 중규모 소규모 네트워크에 적합하다.

각 마디는 NetworkID와 HostID로 나뉘며 HostID가 가질 수 있는 수가 해당 네트워크가 가질 수 있는 ip의 숫자라고 생각해도 된다.

HostID의 첫 주소는 NetworkID로 개인 주소로 할당이 불가하며 마지막 주소는 BroadcastIP로 송출용 IP가 되어 사용이 불가능하다.

A클래스의 경우 NetworkID 가 첫 마디 , B 클래스의 경우 두번째 마디 까지 C클래스의 경우 세번째 마디 까지로 정해진다.

## 사이더 표기법

IP 주소 뒤에 /24 와 같이 붙어 있으면 해당 주소의 Network ID 와 Host ID 를 구분짓는 경계가 24번째 자릿수라는 뜻이 된다. 즉 /24 면 25번째가 첫번째 Host ID 의 첫 자리 수 가 됨.

## 서브넷 마스크 

서브넷 마스크란 AND 연산을 통해 Network ID를 즉각적으로 구할 수 있게 설저된 주소를 뜻한다. 예를 들어 기본적인 C클래스의 서브넷 마스크는 255.255.255.0 이 된다.

## 서브네팅

서브네팅이란 특정 클래스의 서브넷 주소가 남는 경우 이걸 다시 나눠서 사용할 수 있게 만든걸 뜻한다. ex / 한 회사에서 필요로 하는 ip주소가 10개인 경우 C클래스를 할당받으면 굉장히 많이 남기 때문에 C클래스를 나눠서 다른 회사들에게도 할당할 수 있게 한다.

한마디로 IP주소의 낭비를 방지하기 위해 원본 네트워크를 여러개의 서브넷으로 분리하는 과정이다. 

## 예시 

C 클래스 네트워크를 10개의 회사의 주소로 다시 서브넷하는 경우 서브넷 마스크의 HostID의 4자리 수를 추가로 NetworkID로 할당해야 하며 이 경우 서브넷 마스크의 주소는 255.255.255.(128+64+32+16) 이 된다.

이 후 HostID는 남은 4자리수에 첫자리와 마지막 자리를 제외하고 각 10개의 회사에 14개의 IP주소가 할당가능해진다.
